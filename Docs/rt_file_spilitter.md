## 1. 線上作業員 (OP) 操作指南 - 日常操作

本程式在背景自動將測試機台的日誌分割並上傳。您只需要在**換線/換工單時**進行設定。

### 📌 操作步驟

1. **確認畫面狀態**
   - 開啟程式後，畫面應為乾淨的「簡易功能 (OP)」模式。
   - 左上角狀態燈號應為灰色的 `● Idle`。

2. **輸入生產資訊**
   - **WOU (工單號碼)**：點擊欄位，使用條碼槍刷入或手動輸入當前生產工單號碼（例如：`5101-260129012`）。
   - **STA (機台站別)**：確認下拉選單選擇的是您當前所在的機台編號（例如：`10`）。

3. **開始監控**
   - 點擊藍色的 **「▶ Start」** 按鈕。
   - 左上角狀態會變成綠色的 `● Watching`，表示程式已開始自動監控並上傳檔案。
   - 畫面中間會出現綠色的 `✓ OK` 字樣，代表目前運作正常。

4. **停止監控 (換線/下班時)**
   - 點擊黑色的 **「■ Stop」** 按鈕。
   - 狀態燈號變回灰色 `● Idle` 後，即可關閉程式或更換新工單。

### ⚠️ 注意事項
- 如果畫面出現紅色的 `✗ ERR`，或是狀態燈號變成紅色的 `● Error`，請立即**舉手呼叫班長**協助處理。
- 程式執行期間請勿關閉視窗，可以點擊右上角的「−」按鈕將畫面縮小。

## 2. 班長 (Leader) 異常排除指南 - 現場異常處理

當作業員反映程式出現異常時，請依照以下步驟進行初步排除。

### 🔴 常見異常與處理方式

#### 1. 畫面顯示紅色的 `✗ ERR` 或 `● Error`
- **原因**：通常是找不到來源測試日誌檔，或是本機硬碟空間已滿。
- **排除步驟**：
  1. 點擊右上角的「🔒 簡易功能 (OP)」，切換為「🔓 進階功能 (ENG)」。
  2. 檢查 **SRC (來源檔案)** 欄位中的路徑是否正確，檔案是否存在。
  3. 若來源檔案不見了，請確認測試程式是否正常啟動並產生 Log。

#### 2. 檔案一直沒有上傳 / 累積在 Queue 中
- **原因**：網路斷線或遠端伺服器異常。
- **排除步驟**：
  1. 切換至「🔓 進階功能 (ENG)」模式。
  2. 找到 **DST** 欄位旁邊的 **「Test」** 按鈕，點擊它進行網路與連線測試。
  3. 觀察旁邊的燈號：
     - 🟢 **綠燈**：連線正常，程式會自動恢復上傳。
     - 🔴 **紅燈**：連線失敗。請檢查機台網路線、IP 設定，或確認伺服器是否開機。若無法解決請通知產線工程師。

#### 3. 不小心按到停止上傳
- 如果不小心按到了「⏹ Stop Upload」，請先按右上角的「■ Stop」停止整個程式，再重新按下「▶ Start」即可恢復正常運作。

#### 4. 切換顯示模式
- 點擊右上角的 **「🌙」或「☀」** 可以切換深色/淺色主題，可依現場光線調整。

## 3. 產線工程師 (Engineer) 設定與維護手冊 - 系統佈署與設定

此部分說明初次架設機台或系統維護時的詳細設定參數。

### ⚙️ 進入進階模式
點擊畫面上方的「🔒 簡易功能 (OP)」按鈕，展開進階設定面板 (ENG Mode)。

### 📂 路徑與參數設定

- **SRC (Source)**：
  - 測試程式產生的「總日誌檔」路徑（例如 `C:\Test\Log_ALL.txt`）。
  - 請點擊右側資料夾圖示選擇檔案。
- **OUT (Output)**：
  - 本機分割檔案的儲存目錄。
  - 程式會自動在此目錄下依照站別 (`STAxx`) 與工單建立子資料夾存放 `.txt` 檔。
- **DST (Destination)**：
  - 遠端 SFTP 上傳路徑。
  - 格式必須為：`使用者名稱@伺服器IP:/遠端絕對路徑/` 
  - 範例：`root@192.168.100.1:/run/media/nvme0n1p1/rawlogs/`
  - *註：如果此欄位留空，程式將只會把檔案分割存放在 OUT 目錄，不會執行上傳動作。*
- **輪詢間隔 (SpinBox)**：
  - 預設為 `60 s`。代表每隔幾秒檢查一次 SRC 檔案是否有更新。

### 🔑 SSH 連線測試與認證
設定好 DST 後，請務必點擊 **「Test」** 進行連線測試：
- 程式內建自動驗證機制：
  1. 會先嘗試免密碼 (Empty password) 登入。
  2. 若失敗，會檢查 `~/.ssh/` 下是否有金鑰 (`id_ed25519`, `id_rsa` 等)。
  3. 若無金鑰，程式會**自動產生 RSA Key pair** 並嘗試使用 Public Key 登入。
- 測試結果會顯示在 Activity Log 中，請依據 Log 提示排除連線問題（如伺服器端未配置 Authorized Keys 等）。

### 💾 設定檔儲存與轉移
- **儲存設定**：點擊下方的 **「儲存設定」**，目前畫面上的所有參數（包含站別、路徑、工單）會寫入 Windows Registry (QSettings)，下次開啟程式時會自動載入。
- **匯出/匯入設定**：
  - 點擊 **「匯出設定」** 將參數存成 `.json` 檔案。
  - 在新機台架設時，可直接使用 **「匯入設定」** 載入相同的 `.json` 檔，只需修改 `STA` 站別即可快速完成部署。

### 📝 檔案命名與資料結構
程式透過正則表達式解析 Log，產生的檔案命名規則如下：
1. **依站別目錄 (`OUT/STA{id}/`)**：
   - 檔名：`STA{站別}_{日期}_{時間}_{MAC1}_{MAC2}_{結果}.txt`
   - 範例：`STA10_20260129_153000_001A2B3C4D5E_001A2B3C4D5F_PASS.txt`
2. **依工單目錄 (`OUT/{工單號}/` 及遠端 `DST/../{工單號}/`)**：
   - 檔名：`{工單}_{日期}_{時間}_{MAC1}_{MAC2}_{結果}.txt`
   - 範例：`5101-260129012_20260129_153000_001A2B3C4D5E_001A2B3C4D5F_PASS.txt`

程式有防呆機制，若 MAC 位址格式錯誤或結果非 `PASS/FAIL/STOP`，該區段 Log 將會被標記為 Skipped 並跳過處理。
